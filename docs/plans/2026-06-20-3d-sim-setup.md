## 0. Goal & scope

Turn the current **flat 2D** SmoothRide env + demo into a **true 3D simulation**:
agents drive on real San Francisco terrain (hills), perceive their surroundings in
3D **including buildings that occlude line-of-sight**, and the demo shows meshed
cars driving around a 3D map of SF in the browser.

This plan implements the design locked in `session.md` with three keystone
decisions confirmed:

| Decision | Choice |
|---|---|
| **Where 3D lives** | Extend the fast **JAX kinematic env** to 3D — add `z`/grade + **geometric FOV-occlusion** against OSM buildings. Stays fast, scales to 100s–1000s of cars, no GPU. |
| **Primary render** | **CesiumJS** browser viewer (Cesium World Terrain + Cesium OSM Buildings + meshed cars), **offline replay** of logged poses. |
| **Locomotion / Isaac PhysX** | **Deferred** to a separate workstream (Config 2's second policy). Setpoint interface is preserved so it can attach later. |

### In scope
1. Elevation + 3D geometry ground truth (DEM → node `z` + per-edge grade; OSM buildings).
2. 3D-aware env: grade dynamics, `z` in state/trace, 3D neighbor obs.
3. 3D perception: FOV + **building occlusion** (a building hides the car behind it).
4. CMDP reframe: reward = travel time; crash/off-road/rule = **dense per-agent constraint cost**.
5. Deterministic, trace-based **verifier** + a reproducible **trace** schema.
6. **Cesium** 3D demo: meshed cars on 3D SF terrain + buildings, offline replay.

### Out of scope (explicit non-goals here)
- Isaac PhysX rigid-body execution + locomotion controller (later workstream; scaffolding in `smoothride/demo/isaac/` + `smoothride/lowlevel/` stays as-is).
- Cosmos / Antim integration (out-of-loop, separate workstream).
- Photoreal (Cosmos-Transfer) render.
- Full 3D multi-lane junction meshing (we keep flat-on-terrain road ribbons; see Risks).

## 1. Architecture — the 3D pipeline

```
                          BUILD ONCE (host, offline)
  OSM bbox ─┬─ map_loader  → RoadNetwork (x,y)  ──────────────┐
            ├─ terrain.py  → node z + per-edge grade (py3dep) │→ RoadNetwork3D
            └─ buildings.py→ footprints + heights (osmnx)     │   + BuildingSet
                                                              ▼
              routing.py → RoutePool (xy, z, grade, node, junc, lanes, speed)
                                                              ▼
  ────────────────────────────── PER STEP (JAX, vectorized) ──────────────────
   obs = perceive( spatial-hash neighbors  ∧  LINE-OF-SIGHT vs buildings,
                   own kinematics, grade, route_remaining )      ← 3D perception
   action = policy(obs)                         (decentralized, local obs only)
   pos'  = grade-aware bicycle dynamics(pos, action)            ← 3D dynamics
   cost  = check(crash, off-road, rule)         (CMDP constraint channel)
   reward= -Δtravel_time (+ arrival bonus)      (efficiency objective)
   TRACE.append(per-car: t,id,x,y,z,heading,speed,steer,lane,action,events)
  ─────────────────────────────────────────────────────────────────────────────
                                                              ▼
   VERIFIER (pure fn over trace) → per-car arrived?/time?/valid? + costs
                                                              ▼
   export_cesium.py → (x,y,z)→(lon,lat,height) → trajectories.json
                                                              ▼
   CesiumJS (terrain + OSM buildings + meshed cars) → offline replay demo
```

## 2. Phases

Each phase is independently testable and ends in a **visible artifact**
(the `REALISM.md` rule). File paths are concrete; new files marked **NEW**.

### Phase 0 — Scaffolding & deps
- **`pyproject.toml`**: add to the `data` extra: `py3dep`, `rasterio` (DEM);
  `shapely`, `geopandas`/`trimesh` are already pulled via `osmnx` — confirm.
  Add a `viz3d` extra note (Cesium is browser-side via CDN + an **ion access token**, no Python dep).
- Create `data_cache/` entries plan: `sf_dem.tif`, `sf_buildings.parquet` (cached like the graphml).
- **Deliverable:** `pip install -e ".[data]"` succeeds; deps import.

### Phase 1 — Elevation + buildings (3D ground truth)
**NEW `smoothride/data/terrain.py`**
- `load_dem(bbox, resolution=10) -> xarray` via `py3dep.get_dem` (cache to `data_cache/sf_dem.tif`); 10 m default (1 m optional for crisp ridgelines).
- `sample_node_z(net, dem) -> (N,)`: bilinear-sample DEM at each `net.node_xy` (reproject metric→DEM CRS).
- `reconcile_junction_z(...)`: ensure each shared node gets **one** `z` (roads tear at junctions otherwise — `terrain-dem.json` risk).
- `edge_grade(net, node_z) -> (E,)`: `grade = (z_v - z_u) / horizontal_length`; **smooth/clip** (e.g. ±0.35) against DEM-vs-GPS noise.

**NEW `smoothride/data/buildings.py`**
- `load_buildings(bbox) -> GeoDataFrame` via `ox.features_from_bbox(tags={"building": True})` (cache to parquet).
- `height_of(row)`: `height` if present else `building:levels * 3.0` else default (e.g. 8 m).
- `to_metric_polygons(net, gdf) -> BuildingSet`: reproject footprints into the **same shifted-UTM metric frame** as the road net; output (a) polygon vertex arrays + base `z` (from DEM) + height for **render**, and (b) flattened **edge segments** `(S, 2, 2)` for **occlusion** (Phase 3).

**Extend `smoothride/data/map_loader.py`**
- Add `node_z: (N,)` and `edge_grade: (E,)` to `RoadNetwork` (default zeros when terrain disabled, so 2D path still works).
- New `load_road_network_3d(**kw)` convenience that composes graph + terrain + buildings.

- **Tests** (`tests/test_terrain.py`): SF z within plausible range (~0–280 m); at least one edge grade > 0.15 (SF hills are real); buildings GeoDataFrame non-empty; all footprints inside bbox; junction nodes single-valued in z.
- **Artifact:** a quick matplotlib hillshade + building-footprint overlay PNG (`scripts/preview_terrain.py`).

### Phase 2 — RoutePool + Env carry z & grade
**Extend `smoothride/env/routing.py`**
- Add `z: (P, W)` (node z along route) and `grade: (P, W)` (grade of edge leaving wp) to `RoutePool`, filled in `build_route_pool` from `net.node_z`/`net.edge_grade`.

**Extend `smoothride/env/kinematic.py`**
- `Env`: add `routes_z`, `routes_grade` arrays (wired in `make_env`).
- **Grade-aware dynamics** in `step`: uphill reduces effective `accel_max` / `vmax`; downhill lengthens braking (lower effective decel). Keep it a cheap scalar modifier — physics realism stays in the (deferred) Isaac layer; this is the "grade appears in both layers" case from `REALISM.md`.
- Carry **`z`** through `State` (sampled from route by `wp_ptr`) so the trace/export has real height. Dynamics integration stays in the `(x,y)` plane (2.5D) — `z` is draped, not simulated.
- Add **grade scalar** to obs → `obs_dim` becomes `6 + 1 + 4K + 3 + 1`. **Note:** changing `obs_dim` invalidates existing checkpoints → policy must be retrained (call out in DEVLOG).

- **Tests** (`tests/test_dynamics_3d.py`): closed-form checks still hold on flat ground (`s=½at²`, arc radius `R=wheelbase/tanδ`, limit clamps); uphill grade reduces achieved speed vs flat for identical action.

### Phase 3 — 3D perception: FOV + building occlusion
**NEW `smoothride/env/perception.py`** (or extend `spatial.py`)
- Reuse the spatial-hash candidate set (`_candidates`) for neighbors — already O(N·C).
- **Occlusion test:** for each (ego, neighbor) pair, the line segment ego→neighbor is **blocked** if it intersects any building edge segment. Vectorized ray-segment intersection in JAX. **Cull** building segments by the same uniform grid so each car only tests buildings in its ~3×3 cell block (keeps it O(N·B_local)).
- Occluded neighbors are **dropped** from the K-nearest obs (set invalid), so the policy genuinely can't "see through" buildings.
- Add nearest-building distance to obs (and feed Phase 4's off-road/clearance cost).

- **Tests** (`tests/test_occlusion.py`): plant a building between two cars → far car absent from obs; remove building → present; vectorized result matches a brute-force reference within tolerance.
- **Artifact:** top-down PNG with FOV cones + a shadowed (occluded) car behind a building.

### Phase 4 — CMDP reframe + trace + deterministic verifier
**Reward/constraint split in `smoothride/env/kinematic.py`**
- **Reward** = efficiency: `-Δtravel_time` per step + arrival bonus (replaces the dense weighted-sum; keep progress as a shaping term if needed for warm-up).
- **Constraint cost channel** (returned in `info`, per-agent, per-step): `crash`, `off_road`, `rule_violation` as **dense costs on the offending transition** (session Pushback 1 — never discard the whole run). The existing **PPO-Lagrangian** machinery in `smoothride/rl/` consumes this channel (adaptive λ); reuse it.
- **Episode**: fixed `max_steps` horizon + **per-agent done-on-arrival** (session Pushback 2 — bounded rollouts; one slow car can't hold the batch open). Continuous-respawn throughput metric preserved.

**NEW `smoothride/env/trace.py`**
- `Manifest`: `run_id, seed, scenario_id, policy_checkpoint_id, config_hash` (env params + map version) → replayable bit-for-bit.
- `Timeline` (per car, per step): `t, id, x, y, z, heading, speed, steer, lane, action, wp_ptr, dist_remaining`.
- `Events`: crash (who/where/closing-speed), off_road, rule_violation, arrival (travel_time), respawn.
- Writer → **JSONL** under `runs/traces/<run_id>.jsonl` (+ aggregate summary).

**NEW `smoothride/eval/verifier.py`** — *the key requirement*
- **Pure function over the trace** — no re-sim, no randomness, no wall-clock, no network/LLM. Same trace → same verdict.
- Geometric predicates: `crash` = min footprint dist < `collision_radius` at any step; `off_road` = car center outside the drivable road polygon (buffered OSM centerlines); `rule_violation` = speed > edge limit / wrong-way on one-way / uncontrolled-junction yield broken; `arrived` = reached destination wp within horizon; `valid_run` = all constraints satisfied for all cars at all steps.
- Emits per-car `{arrived, travel_time, valid, costs:{crash,off_road,rule}}` + run aggregates (throughput, mean/median travel time, crash count, validity flag).

- **Tests** (`tests/test_verifier.py`): determinism (same trace twice → identical verdict); hand-crafted traces — a planted crash, an off-road excursion, a clean run — each verified correctly; verifier verdict matches the env's online `info` on a real rollout.

### Phase 5 — 3D trace export + Cesium viewer
**NEW `smoothride/demo/export_cesium.py`** (mirrors `export_web.py` but 3D)
- Replay checkpoint → trace (with `z` draped from terrain).
- Reproject `(x,y)`→`(lon,lat)` (reuse `_lonlat_transformer`/`_to_lonlat`) and carry **`height`** (`z`).
- Pack `trajectories.json`: per-car `lng/lat/height/hdg/spd/crash`, meta (`dt`, `n_steps`, center, bounds), verifier summary (trips, crashes, valid). Reference Cesium ion assets for terrain + OSM Buildings (no geometry baked into JSON).

**NEW `smoothride/demo/cesium/`** (`index.html`, `app.js`, `config.example.js`, `README.md`)
- CesiumJS via CDN; `Cesium.createWorldTerrainAsync()` + `Cesium.createOsmBuildingsAsync()`.
- Cars as glb or procedural mesh (reuse `carmesh.js` silhouette) placed via `SampledPositionProperty` / CZML at `(lon, lat, height)` with `heading`, time-dynamic playback + clock.
- HUD: trips completed, crashes, **valid-run** flag, clock — driven by the verifier summary.
- Graceful fallback + clear "set your ion token in `config.js`" message (mirror the Mapbox-token pattern in the existing `app.js`).

- **Artifact:** screen recording + stills of meshed cars driving 3D SF (hills + buildings). This is the headline deliverable.

### Phase 6 — Validation, wiring, docs
- Extend **`scripts/smoke_env.py`** to cover the 3D env (vmap shapes with new `obs_dim`, grade + occlusion paths run under `jit`).
- **`scripts/`** one-shot: build 3D ground truth → short rollout → trace → verify → export Cesium, so the whole pipeline runs end-to-end with one command.
- Update **`DEVLOG.md`** (new entries per phase), **`components/`** (`terrain-dem`, `road-mesh-builder` → status, add a `cesium` viz component, update `viz-demo`), and **`STACK.md`** stack table.

## 3. Risks & mitigations

| Risk | Mitigation |
|---|---|
| DEM resolution vs OSM GPS noise → noisy/huge grades | smooth + clip grades (±0.35); 10 m DEM default |
| Junction z mismatch tears roads | `reconcile_junction_z` — one z per shared node |
| Missing building heights | fallback `levels×3` then default; cosmetic only |
| Occlusion cost at 1000s of cars | grid-cull buildings to local cells; vectorized ray-segment; cap segments/cell |
| `obs_dim` change invalidates checkpoints | expected — retrain; document; keep a 2D-compat flag for old runs |
| Cesium needs an ion access token | free token; `config.example.js` + graceful fallback message |
| Off-road predicate needs a drivable polygon | buffer OSM centerlines by lane-width once (reuse `road-mesh-builder` buffering idea), share with verifier |
| Scope creep into Isaac/Cosmos | hard-deferred; setpoint interface preserved so they attach cleanly later |
